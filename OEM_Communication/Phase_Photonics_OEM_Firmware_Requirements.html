Date: February 5, 2026

Hardware: Phase Photonics OEM with Custom LED Controller

1. LED Control Functions

Individual Channel Control

turn_on_channel(channel: 'a'|'b'|'c'|'d')
    Description: Enable specific LED channel
    Parameters: channel - LED channel identifier ('a', 'b', 'c', or 'd')
    Returns: bool (success/failure)

set_intensity(channel: str, value: 0-255)
    Description: Set LED brightness for specific channel
    Parameters:
        - channel: LED channel identifier
        - value: Intensity value (0-255 range)
    Returns: bool (success/failure)
    Note: Channel must be enabled first via turn_on_channel()

turn_off_all_channels()
    Description: Disable all LED channels simultaneously
    Returns: bool (success/failure)

Batch Operations (CRITICAL - 15x faster)

set_batch_intensities(a: int, b: int, c: int, d: int)
    Description: Atomic update of all 4 LED channels simultaneously
    Parameters:
        - a: Intensity for channel A (0-255)
        - b: Intensity for channel B (0-255)
        - c: Intensity for channel C (0-255)
        - d: Intensity for channel D (0-255)
    Command Format: batch:<a>,<b>,<c>,<d>
    Example: batch:255,128,0,200
    Returns: bool (success/failure)
    Performance: 15x faster than sequential commands
Implementation Note: Channel Enable Prerequisite - LEDs must be "enabled" via turn_on_channel() before intensity takes effect. The batch command handles this automatically.

2. Detector Read Functions

read_intensity(data_type: np.uint16) -> np.array[1848]
    Description: Read single spectrum from detector
    Returns: Array of 1848 pixels (13-bit ADC)
    Specifications:
        - Pixel count: 1848
        - ADC resolution: 13-bit
        - Max count: 8191 (saturation threshold)
        - Dark current baseline: ~900 counts
    Timing: ~Integration_Time × 1.93

read_intensity_batch(num_scans: int) -> np.array[1848]
    Description: Software averaging - read num_scans and average
    Parameters: num_scans - Number of scans to average (typical: 8)
    Returns: Averaged spectrum array [1848 pixels]
    Timing: Integration_Time × 1.93 × num_scans
    Example: 8 scans at 100ms integration = ~250ms total
CRITICAL: Hardware averaging is currently broken in firmware - it ignores integration time and uses ~1ms instead. MUST use software loop (read multiple times and average in software).

3. Integration Time Control

set_integration(integration_ms: float)
    Description: Set detector integration time in milliseconds
    Parameters: integration_ms - Integration time (10-1000ms typical)
    Returns: bool (success/failure)
    Usage: Required for signal optimization during calibration

set_trigger_timeout(timeout_ms: int)
    Description: Set detector read timeout
    Parameters: timeout_ms - Timeout in milliseconds
    Recommended: 2-3x integration time
    Returns: bool (success/failure)

4. Wavelength Calibration

read_wavelength() -> np.array[1848]
    Description: Return calibrated wavelength array
    Returns: Array of wavelengths (536-726 nm range)
    Source Priority:
        1. External calibration override (if available)
        2. EEPROM polynomial calibration
    Format: 4-degree polynomial [c0, c1, c2, c3]

read_calibration_coefficients() -> [c0, c1, c2, c3]
    Description: Return raw EEPROM calibration polynomial coefficients
    Returns: List of 4 float coefficients
    Usage: Backup if external override not available
    Polynomial: wavelength(pixel) = c0 + c1*pixel + c2*pixel² + c3*pixel³

5. Servo Control Functions

Mode Switching (Runtime Operations)

set_mode(mode: 's'|'p')
    Description: Switch polarizer to S-mode or P-mode
    Parameters: mode - 's' for S-polarization, 'p' for P-polarization
    Commands:
        - 'ss\n' - Switch to S-mode
        - 'sp\n' - Switch to P-mode
    Behavior: Reads calibrated positions from EEPROM and moves servo
    Settling time: 400ms
    Returns: bool (success/failure)

Direct Position Control

servo_move_raw_pwm(pwm: 0-255)
    Description: Move servo to PWM position (0-255 maps to 5-175°)
    Parameters: pwm - PWM value (0-255)
    Conversion: angle = 5 + (pwm/255.0) × 170
    Command Format: 'servo:{angle},500\n'
    Example: PWM=127 → angle=90° → command='servo:90,500\n'
    Duration: 500ms pulse + 100ms settling
    Returns: bool (success/failure)

servo_move_calibration_only(s_pos: int, p_pos: int)
    Description: FOR CALIBRATION ONLY - move to test positions
    Parameters:
        - s_pos: S-position PWM (0-255)
        - p_pos: P-position PWM (0-255)
    Command Format: 'sv{s:03d}{p:03d}\n'
    Example: servo_move_calibration_only(10, 100) → 'sv010100\n'
    Warning: For calibration sweep only, not runtime use
    Returns: bool (may return True even without ACK)

Servo Configuration

set_servo_speed(speed_ms: int)
    Description: Set servo pulse duration (movement speed)
    Parameters: speed_ms - Pulse duration in milliseconds
    Range: 200-2000ms
        - 200ms = fastest
        - 500ms = default
        - 2000ms = slowest
    Command Format: 'servo_speed:{speed_ms}\n'
    Returns: bool (success/failure)

set_servo_positions(s_pos: int, p_pos: int)
    Description: Store calibrated S and P positions in EEPROM
    Parameters:
        - s_pos: Calibrated S-mode position (0-255 PWM)
        - p_pos: Calibrated P-mode position (0-255 PWM)
    Usage: Called after calibration complete
    Persistence: Saved to EEPROM for future use
    Returns: bool (success/failure)

get_servo_positions() -> dict[str, int]
    Description: Retrieve stored S and P positions from EEPROM
    Returns: Dictionary with keys 's' and 'p'
    Example: {'s': 85, 'p': 170}
    Usage: Read at startup for runtime mode switching

6. Device Management

initialize(serial_number: str) -> handle
    Description: Connect to OEM device by serial number
    Parameters: serial_number - Device serial (e.g., "ST00007", "ST00011")
    Returns: Device handle for subsequent operations
    Raises: Exception if device not found or connection fails

deinitialize(handle)
    Description: Clean disconnect from device
    Parameters: handle - Device handle from initialize()
    Behavior: Releases USB resources and closes connection
    Returns: bool (success/failure)

get_device_info() -> dict
    Description: Query device information and capabilities
    Returns: Dictionary containing:
        - serial: Device serial number
        - firmware_version: Firmware version (e.g., "4.2")
        - pixel_count: Number of pixels (1848)
        - wavelength_range: Tuple (min_nm, max_nm) = (536, 726)
        - adc_resolution: Bit depth (13)
        - max_count: Saturation threshold (8191)

8. Critical Operational Sequences

Standard Acquisition (4-step)

1. oem.set_integration(integration_ms)
   # Set detector integration time

2. oem.set_batch_intensities(a=led_a, b=led_b, c=led_c, d=led_d)
   # Atomically set all LED channels

3. time.sleep(0.045)
   # Wait 45ms for LED stabilization

4. spectrum = oem.read_intensity_batch(num_scans=8)
   # Read and average 8 scans

LED Convergence/Calibration Workflow

# Iterative convergence to target signal (40% of max counts = ~3276)
for iteration in range(15):
    # 1. Set current LED values
    oem.set_batch_intensities(a, b, c, d)

    # 2. Measure signal
    spectrum = oem.read_intensity_batch(num_scans=8)
    signal = np.max(spectrum)

    # 3. Check saturation (ZERO saturated pixels required to pass)
    if signal >= 8191:
        # Reduce LED intensity proportionally
        led_values = led_values * 0.9
        continue

    # 4. Adjust LEDs based on measured signal vs target
    target_signal = 3276  # 40% of max
    adjustment_factor = target_signal / signal
    new_led = current_led * adjustment_factor

    # 5. Check convergence
    if abs(signal - target_signal) < tolerance:
        break  # Converged successfully

Servo Calibration Workflow

# Full servo calibration to find S and P positions
angles = []
intensities = []

# 1. Sweep servo across full range
for test_angle in range(5, 176, 5):  # 5° to 175° in 5° steps
    # Convert angle to PWM
    pwm = int((test_angle - 5) * 255 / 170)

    # Move servo to test position
    oem.servo_move_calibration_only(s=pwm, p=pwm)
    time.sleep(0.4)  # Settling time

    # Measure spectrum
    spectrum = oem.read_intensity_batch(num_scans=8)
    intensity = np.max(spectrum)

    angles.append(test_angle)
    intensities.append(intensity)

# 2. Analyze results
s_pos_angle = angles[np.argmax(intensities)]  # Maximum signal = S-mode
p_pos_angle = angles[np.argmin(intensities)]  # Minimum signal = P-mode

# 3. Convert angles back to PWM
s_pos_pwm = int((s_pos_angle - 5) * 255 / 170)
p_pos_pwm = int((p_pos_angle - 5) * 255 / 170)

# 4. Store calibrated positions
oem.set_servo_positions(s_pos_pwm, p_pos_pwm)

10. Critical Design Principles

  1. Batch LED Commands Preferred: Use set_batch_intensities() instead of sequential calls (15x performance improvement)
  2. Channel Enable Prerequisite: Must call turn_on_channel() BEFORE set_intensity() works (V2.0+ firmware requirement)
  3. Zero Saturation Requirement: LED calibration passes ONLY with zero saturated pixels (no pixels at 8191 counts)
  4. Phase Photonics Isolation: Keep separate API from USB4000 (1848 pixels vs 3700 pixels - prevents mixing)
  5. Software Averaging Only: Hardware averaging is broken - always use software loop (read N times and average)
  6. Timing Multiplier: Total read time = Integration_Time × 1.93 × num_scans
  7. Model-Based Convergence: Use ML predictions + linear slope adjustment for faster LED convergence
  8. Servo Settling Time: Always wait 400ms after servo move before reading spectrum
  9. EEPROM vs Runtime: Calibrated servo positions stored in EEPROM at startup, never modified during runtime
  10. Single Measurer Pattern: USB spectrometer requires exclusive access (no parallel operations)

11. Communication Protocol Summary

LED Commands

  • la\n, lb\n, lc\n, ld\n - Turn on single LED channel
  • lx\n - Turn off all channels
  • b<ch><intensity>\n - Set intensity (e.g., ba255 = LED A at full brightness)
  • lm:a,b,c,d\n - Enable multiple channels
  • batch:<a>,<b>,<c>,<d>\n - Batch intensity (firmware V1.1+)

Servo Commands

  • ss\n - Switch to S-mode (reads S-position from EEPROM)
  • sp\n - Switch to P-mode (reads P-position from EEPROM)
  • sv{s:03d}{p:03d}\n - Calibration move (e.g., sv010100)
  • servo:{angle},500\n - Direct angle move (e.g., servo:90,500)
  • servo_speed:{speed_ms}\n - Set pulse duration (e.g., servo_speed:500)

Response Protocol

  • Standard ACK: b'6' or b'1'
  • Empty response acceptable on V1.1 firmware
  • Response timing: typically within 100-150ms
  • Movement time: 500ms pulse + 100ms settling = 600ms total
