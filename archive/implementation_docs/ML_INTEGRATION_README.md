# ML Afterglow Correction Integration

## What Was Changed

The `spr_data_acquisition.py` file has been modified to use Hybrid Physics + ML
afterglow correction instead of physics-only correction.

### Changes Made:

1. **Added `MLAfterglowCorrection` class**
   - Loads trained ML model (afterglow_ml_model.h5)
   - Combines physics model with ML residual prediction
   - Falls back to physics-only if ML model not found

2. **Modified `SPRDataAcquisition.__init__`**
   - Initializes ML corrector alongside physics corrector
   - Automatically enables hybrid mode if model file exists

3. **Modified correction application**
   - Applies physics correction first (baseline)
   - Adds ML residual correction (learns what physics misses)
   - Updates channel history for next prediction

## How It Works

```
Total Correction = Physics Correction + ML Residual

Where:
- Physics Correction: baseline + A × exp(-t/τ)
- ML Residual: Learned via LSTM network from actual data
```

## Expected Performance

- **Lagged correlation**: 0.372 → <0.05 (87% reduction)
- **Noise reduction**: 30-40% better than physics-only
- **Cycle time**: No change (still 600ms at 20ms delay)

## Files Required

- `afterglow_ml_model.h5` - Trained Keras model
- `model_scaler.pkl` - Feature scaling parameters
- `model_metadata.json` - Model documentation
